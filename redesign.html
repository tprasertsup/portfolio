Responsive Redesign

Identifying Usability Problems

I chose the website of my cousin's dental clinic. The clinic is based in Thailand, so most of the texts are in Thai.

I extend this project to fulfill the capstone requirement by redesigning the whole website.

Usability Problems

  1. Not responsive, e.g., an overflow text outside the navigation bar in mobile mode
  2. Poor hierarchy, e.g., subheading – dental category – in Team page does not create a sense of higher in a hierarchy to the dentist names
  3. Unclickable phone numbers in Home and Contact pages
  4. Hard to compare prices on the Service page
  5. Too much text on the About and Knowledge page can be overwhelmed, and hard to understand the main points

Accessibility Problems

  1. Dense and small text
  2. Lack of contrast between white texts and light green background
  3. No alternative text in some images
  4. Narrow letter spacing in heading typeface making it hard to read

Visual Redesign

I chose to redesign the website to be one-page for a natural interaction and a better mobile experience, since many users, these days, search up the clinic using mobile phones and tablets.

Some important design that I want to raise are:

  1. A grid should be used instead of a flexbox. Since my design haves multiple sections on one page, a row layout is enforced to separate different parts. Also, a column layout is for organizing elements in each section, shortening the website's length.
  2. For About, Team, Knowledge, and Contact sections, to avoid having small images and texts, the number of columns decreases as the window size decreases.
  3. If the window size is smaller than normal desktop's, the navigation bar will transform into a dropdown menu. Still, in both cases, the menu bar always stays at the top of the screen.
  4. While no component in the website changes the page, elements in Team and Knowledge sections will open a modal (an overlay). Also, buttons in Contact section will open the link in the new tab, instead of changing the page.
  5. Every element that is clickable will change thier appearance when they are hovered or clicked. Please see the Visual Design Style Guide below for more information.

Responsive Redesign

Here are some observations about the website:

  1. The redesigned website works in any window size and even if the font size of the browser is changed.
  2. Even though the website is in a single page, it takes around 4 seconds to scroll down to the bottom in a desktop, and 8 seconds in a mobile.
  3. After I test the website with WebAIM WAVE, the only persisting issues are low contrast between brand color (00B1B1) and white background. I did not fix this because I want to reinforce the brand.
