Creating Hypotheses for Statistical Tests

We will run statistical tests for these two metrics: Time to Completion and Return Rate.

Time to Completion

In this case, we will measure the time each user uses for ordering cacti. Since we believe that the website's variant(B) design will improve user experience, i.e., decreasing the time users spend on the website, we achieve the following null and alternative hypotheses.

Null hypothesis: Both versions of the website will have the same ordering time.

Alternative hypothesis: The variant(B) version of the website will decrease the amount of time needed to complete the order.

Return Rate

In this case, we will measure the number of users who revisit the ordering page after they checkout. Since we believe that the website's variant(B) design will improve user experience, we assume that users are more likely to return to the ordering page if they visit version B. Therefore, null and alternative hypotheses for the return rate are below.

Null hypothesis: Both versions of the website will have the same return rate.

Alternative hypothesis: The variant(B) version of the website will increase the return rate.

Collecting Data and Applying Scripts

For this A/B test, I asks people, both in the UIUX class and outside the class, to perform the task of filling the cart with at least $150 of catci/succelents. Thus, there was a wide range of users - from teenagers to middle-aged people, those with programming knowledge to non-CS concentrators, and people interested in cacti to those who do not. After two days of data collection, 44 people had used the website. 26 of them saw version A, while the other 18 saw version B.

Conclusions and Takeaways

Below is the infographic describing the results and conclusions from each statistical test.

Takeaways

The results do not show any statistical significance, possibly because

  1. The sample size is too small (~40 users in total), so a few edge cases can greatly impact the results.
  2. The users are varied in ages, interests, and knowledge. They were not assigned to see a specific version of the website. Thus, middle-aged people, who tend to spend more on reading a website, might all see version A, for example.
  3. Most users did the task as they were asked to. Hence, they might focus only on completing the task, and the experiment was not valid and practical.

Since there is no statistical difference between these two designs, we cannot draw any conclusion about design principle without having more data.
